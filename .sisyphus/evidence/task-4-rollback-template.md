# 高风险操作回滚需求模板

## 概述

本模板用于规范所有涉及数据库、Cloudflare R2、远程 API 或生产环境的高风险操作。每个高风险计划必须填写以下所有字段，否则计划不予通过。

---

## 强制字段清单

### 1. 备份前提 (Prerequisites)

**字段说明**: 执行操作前必须完成的备份步骤

**要求内容**:
- [ ] 列出所有需要备份的数据源
- [ ] 明确备份方法（导出文件、快照、版本控制等）
- [ ] 验证备份完整性

**示例**:
```
- 数据库: 导出 D1 数据库完整快照到本地
- R2: 列出当前所有文件，保存文件列表到 backup-r2-files.json
- 配置: 导出当前环境变量到 .env.backup
```

---

### 2. 干运行验证 (Dry-run)

**字段说明**: 在不实际修改生产环境的情况下验证操作

**要求内容**:
- [ ] 列出所有 dry-run 检查点
- [ ] 明确如何验证模拟操作的正确性
- [ ] 定义 dry-run 通过标准

**示例**:
```
- 使用 --dry-run 参数运行脚本
- 检查输出日志中所有文件路径是否正确
- 验证数据库查询结果与预期一致
```

---

### 3. 批量分段执行 (Batch Execution)

**字段说明**: 将大规模操作拆分为小批次，降低单次风险

**要求内容**:
- [ ] 明确批次大小（每批处理数量）
- [ ] 列出所有批次执行顺序
- [ ] 每批次之间设置检查点

**示例**:
```
- 数据库更新: 每批 10 条记录，共 30 批
- R2 文件上传: 每批 5 个文件，分批执行
- 每批次完成后验证数据完整性
```

---

### 4. 失败停止条件 (Failure Criteria)

**字段说明**: 定义何种情况应立即停止操作并触发回滚

**要求内容**:
- [ ] 列出所有失败场景
- [ ] 定义每种失败场景的判定标准
- [ ] 明确停止后的处理流程

**示例**:
```
- 任何一条记录更新失败 → 立即停止
- R2 上传返回错误 → 立即停止
- 数据库连接超时 → 立即停止
- 超过 5% 失败率 → 立即停止
```

---

### 5. 恢复步骤 (Recovery Steps)

**字段说明**: 执行回滚的具体操作步骤

**要求内容**:
- [ ] 按顺序列出每一步恢复操作
- [ ] 每步操作有明确的时间限制
- [ ] 定义恢复完成判定标准

**示例**:
```
1. 停止所有正在执行的操作
2. 使用备份恢复数据库: wrangler d1 execute DB --file=backup.sql
3. 恢复 R2 文件: 重新上传备份文件
4. 验证数据一致性: 对比备份前后的记录数
```

---

### 6. 验证步骤 (Verification Steps)

**字段说明**: 确认恢复成功的检查项

**要求内容**:
- [ ] 列出所有验证点
- [ ] 定义每项验证的通过标准
- [ ] 明确验证责任人

**示例**:
```
- 数据库: 记录数与备份前一致，无数据丢失
- R2: 文件数量与备份前一致，文件可正常访问
- API: 各端点返回正常，无 500 错误
- 前台页面: 关键页面可正常访问
```

---

## 高风险类别定义

### 数据库操作
- 写入/更新/删除数据
- 结构变更（ALTER TABLE）
- 批量数据迁移

### R2 存储操作
- 删除文件
- 移动/重命名文件
- 批量上传/下载

### 远程 API 操作
- 调用第三方支付 API
- 调用认证/授权 API
- 调用外部数据同步 API

### 生产环境操作
- 直接修改生产服务器配置
- 部署到生产环境
- 修改生产环境域名/路由

---

## 审批规则

1. **无回滚计划 = 计划驳回**: 任何高风险操作没有完整的回滚计划，计划不予审批
2. **字段缺失 = 计划驳回**: 上述 6 个强制字段缺一不可
3. **验证不可行 = 计划驳回**: 如果无法定义明确的验证步骤，计划不予通过

---

## 模板使用说明

1. 复制本模板到新计划文档的高风险操作章节
2. 按顺序填写所有 6 个强制字段
3. 确保每个字段的内容具体、可执行、可验证
4. 提交审批前由计划作者自行检查完整性
