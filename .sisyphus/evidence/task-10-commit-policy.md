# 提交策略规范

## 概述

本文档定义了项目统一的 Git 提交规则，确保所有提交信息符合项目语言规范。

## 核心约束

### 语言要求

- **强制使用简体中文**（简体字）
- 禁止使用繁体中文（如"裡""為""發"等）
- 代码注释、提交信息、对话回复全部使用简体

### 允许的提交前缀

| 前缀 | 使用场景 | 示例 |
|------|----------|------|
| `feat:` | 新功能、新特性 | `feat: 添加用户登录功能` |
| `fix:` | Bug 修复、问题修正 | `fix: 修复首页布局错位问题` |
| `chore:` | 维护任务、配置更新 | `chore: 更新依赖包版本` |
| `refactor:` | 代码重构、性能优化 | `refactor: 重构数据获取逻辑` |

## 提交格式

```
<prefix>: <简短描述>

[可选的详细说明]
```

### 格式规则

1. **前缀小写**：使用小写前缀，如 `feat:` 而非 `FEAT:`
2. **描述简洁**：第一行不超过 50 个字符
3. **使用中文**：所有描述文字使用简体中文
4. **动词开头**：描述以动词开头，如"添加""修复""更新"
5. **无需句号**：第一行末尾不加句号

## 正确示例

```bash
# 功能提交
git commit -m "feat: 添加动态详情页"

# 修复提交
git commit -m "fix: 修复移动端导航栏错位"

# 维护提交
git commit -m "chore: 更新项目依赖到最新版本"

# 重构提交
git commit -m "refactor: 重构 API 请求封装逻辑"
```

## 错误示例

```bash
# 错误：使用英文
git commit -m "feat: add new feature"

# 错误：使用繁体中文
git commit -m "feat: 添加用戶登入功能"

# 错误：缺少前缀
git commit -m "添加新功能"

# 错误：使用不允许的前缀
git commit -m "test: 添加单元测试"

# 错误：描述过于模糊
git commit -m "fix: 修复问题"
```

## 文档更新策略

### 纯文档变更

当提交仅涉及文档更新时，使用以下策略：

| 文档类型 | 前缀 | 示例 |
|----------|------|------|
| 代码内注释 | `chore:` | `chore: 更新组件注释说明` |
| 项目文档 | `docs:` | `docs: 更新 API 接口文档` |
| 配置文件 | `chore:` | `chore: 更新部署配置文档` |

### 文档与代码混合变更

如果文档变更是代码变更的附属产物，跟随代码变更使用对应前缀：

```bash
# 添加新功能并更新相关文档
git commit -m "feat: 添加实时直播状态组件

- 支持每15分钟自动刷新
- 更新相关文档说明"
```

## 与 AGENTS.md 的关系

本文档与 `AGENTS.md` 保持一致：

- AGENTS.md 语言规则要求使用简体中文
- AGENTS.md Git 提交规则指定前缀为 `feat:` `fix:` `chore:` `refactor:`
- 本文档是上述规则的具体实现

## 生效范围

- 所有代码提交必须遵循此规范
- Pull Request 标题也需符合此规范
- CI/CD 流水线可能据此验证提交格式
